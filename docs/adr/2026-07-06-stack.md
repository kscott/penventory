# Stack: Node + SvelteKit + SQLite + Drizzle

**Status:** Accepted

**Context:**
The original plan was Rails. The vision/PRD revision session made the visuals-first,
animation-heavy browse experience (`animate:flip`-style shuffling of pens/inks) the actual point
of the app, not an afterthought bolted onto a CRUD backend.

**Decision:**
Node.js (`node:22-slim`) + SvelteKit + SQLite + Drizzle ORM + `colorjs.io` + `sharp` + Zod +
Vitest + Playwright + `pino` + `prom-client`. Full per-layer reasoning in `project-plan.md`'s
Stack table.

**Consequences:**
- SvelteKit's built-in `animate:flip` gives the shuffle/reshuffle interaction natively, no
  separate animation library.
- Drizzle chosen over an ActiveRecord-style ORM — see
  [[2026-07-06-layered-architecture]] for why that matters to testability.
- Old Rails-based `project-plan.md` is superseded in full by the current one.
